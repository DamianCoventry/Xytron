Running A Game

SageEngine requires a command line argument. This argument is the relative path to a game file that was saved from SageStudio. This relative path must point to a file in the Games subdirectory in your Content Directory. Note that game files saved by SageStudio have a "ssg" file extension. The easiest way to create this command line argument is via an Explorer shortcut at the root of your Content Directory.

After having saved my PengiuShooter game and exited SageStudio, my Content Directory looks like this:

So to create my Explorer shortcut I drag SageEngine.exe with the right mouse button to an empty part of the Explorer window, and release the right mouse button. Explorer displays the following popup menu:

I choose Create Shortcuts Here to create the shortcut. Next I right click on the new shortcut and choose Rename, and type in Run PenguinShooter, and press Enter. I then right click on my PenguinShooter shortcut and choose Properties. In the Target editbox, I add to the end of the text the relative path to my PenguinShooter game, which is Games\PenguinShooter.ssg. The dialog should look like this:

Click OK to accept the changes.

Note: If your game's name has a space in it, such as if I had called my game "Penguin Shooter", instead of "PenguinShooter", then I would have needed to put quotes around the relative path, that is, I would have had to add the text "Games\Penguin Shooter.ssg", including the quotes, to the Target edit box within the dialog.

That's it. To run Penguin Shooter, double click the Run PenguinShooter shortcut!
